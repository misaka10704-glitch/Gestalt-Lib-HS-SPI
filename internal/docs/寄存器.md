## 0. 先建立一张地图

你们把整个 Slave 拆成三层，各管一件事：

```
SPI 引脚 (sclk, cs_n, mosi, miso)
        │
        ▼
┌─────────────────────┐
│  spi_controller     │  把「异步 SPI 引脚」翻译成「clk 域里的脉冲」
│  同步 + 边沿 + mode  │
└─────────┬───────────┘
          │ sample_stb, shift_stb, cs_*, mosi_s
          ▼
┌─────────────────────┐
│  spi_shift_register │  只管移位：收 MOSI、发 MISO
└─────────┬───────────┘
          │ byte_done, rx_byte
          ▼
┌─────────────────────┐
│ spi_system_interface│  和用户逻辑握手：rx_valid, tx_load
└─────────────────────┘
```

你觉得 `spi_shift_register` 云里雾里，多半是因为它接了一堆来自上一层的「翻译后的信号」。先把 `spi_controller` 的输出看懂，移位寄存器就清晰了。

## 1. Mode 解耦：移位寄存器不需要知道 CPOL/CPHA

### 问题是什么？

SPI 有 4 种 mode，差别在于：
- **CPOL**：空闲时 SCLK 高还是低
- **CPHA**：在 leading 边沿还是 trailing 边沿采样

如果移位逻辑里写死「上升沿收、下降沿发」，换 mode 就要改移位寄存器——耦合在一起，难维护。

### 解法：controller 把边沿翻译成两个 strobe

```62:72:spi_slave/spi_slave_p1/src/spi_controller.v
    wire sclk_rise =  sclk_s & ~sclk_prev;
    wire sclk_fall = ~sclk_s &  sclk_prev;

    wire leading_edge  = CPOL ? sclk_fall : sclk_rise;
    wire trailing_edge = CPOL ? sclk_rise : sclk_fall;

    wire sample_edge = CPHA ? trailing_edge : leading_edge;
    wire shift_edge  = CPHA ? leading_edge  : trailing_edge;

    assign sample_stb = cs_active & sample_edge;
    assign shift_stb  = cs_active & shift_edge;
```

对移位寄存器来说，世界被简化成两句话：

| 信号 | 含义 |
|------|------|
| `sample_stb` | **现在采样 MOSI，往 RX 移位寄存器塞 1 bit** |
| `shift_stb` | **现在 TX 移位寄存器移 1 bit，准备下一位 MISO** |

`spi_shift_register` 里完全没有 `CPOL`/`CPHA`/`sclk`，只有：

```65:79:spi_slave/spi_slave_p1/src/spi_shift_register.v
            if (sample_stb && cs_active) begin
                rx_shift <= rx_shift_next;
                // ... bit 计数、byte_done
            end

            if (shift_stb && cs_active) begin
                tx_shift <= tx_shift_next;
            end
```

这就是 **mode 解耦**：mode 逻辑全在 controller；移位寄存器是「mode 无关的 bit 泵」。

### Mode 0 下的物理时序（你们 Phase 1 用的）

```
CS_N  ‾‾‾\________________________________/‾‾‾
SCLK  ‾‾‾\_/‾\_/‾\_/‾\_/‾\_/‾\_/‾\_/‾\___
              ↑       ↓       ↑       ↓
         sample_stb  shift   sample  shift
         (收 MOSI)  (发下bit) (收)   (发)
MOSI  ----< B7 >< B6 >< B5 >...
MISO  ----< T7 >< T6 >< T5 >...   (T = TX 要发的位)
```

注意：**收和发不在同一个 SCLK 边沿**。这就是为什么后面要说「全双工需要两根移位寄存器」。

### 还有一个细节：运行在 `clk` 域，不是 `sclk` 域

`sample_stb` / `shift_stb` 不是 SCLK 本身，而是在 **系统时钟 `clk` 的某个上升沿** 打出来的 **单周期脉冲**。

引脚先要双 flop 同步（防亚稳态）：

```35:37:spi_slave/spi_slave_p1/src/spi_controller.v
            sclk_sync <= {sclk_sync[0], sclk};
            cs_sync   <= {cs_sync[0], cs_n};
            mosi_sync <= {mosi_sync[0], mosi};
```

所以整条链路是：

```
sclk 边沿 → 在 clk 域被检测到 → 产生 1 拍 sample_stb/shift_stb → 移位寄存器在下一个 clk 沿动作
```

你写的 `spi_shift_register_rx` 如果每个 `clk` 都移位，就等于**不管 SCLK**，和 SPI 协议对不上。

## 2. CS 帧：一次传输的「开始 / 进行中 / 结束」

SPI 里 **CS（片选）定义了一帧**。你们用三个信号描述帧状态：

```58:60:spi_slave/spi_slave_p1/src/spi_controller.v
    assign cs_active = ~cs_n_s;
    assign cs_start  = cs_n_prev & ~cs_n_s;
    assign cs_end    = ~cs_n_prev & cs_n_s;
```

| 信号 | 何时为 1 | 物理含义 |
|------|----------|----------|
| `cs_active` | CS 为低期间 | 当前在一帧传输里 |
| `cs_start` | CS 下降沿那一拍 | **新帧开始** |
| `cs_end` | CS 上升沿那一拍 | **帧结束** |

### 2.1 新帧开始：`cs_start`

```57:58:spi_slave/spi_slave_p1/src/spi_shift_register.v
            end else if (cs_start) begin
                bit_cnt <= {$clog2(DATA_WIDTH){1'b0}};
```

作用：**bit 计数清零**，从第 0 位重新开始数。

同时 `spi_system_interface` 在 `cs_start` 时加载第一个要发的字节：

```24:24:spi_slave/spi_slave_p1/src/spi_system_interface.v
    wire do_tx_load = tx_ready && (cs_start || (byte_done && cs_active));
```

### 2.2 帧进行中：CS 保持低，连续多字节

testbench T3 就是这种模式：CS 拉低后不抬起，连续发 `0x01, 0x02, 0x03, 0x04`。

```
CS_N  ‾‾\__________________________________/‾‾
         │← 字节1 →│← 字节2 →│← 字节3 →│
```

字节之间**没有** `cs_start`（CS 没再跳变），靠 `byte_done` 衔接：

```68:71:spi_slave/spi_slave_p1/src/spi_shift_register.v
                if (bit_cnt == DATA_WIDTH - 1) begin
                    rx_byte_r <= rx_byte_next;
                    byte_done <= 1'b1;
                    bit_cnt   <= {$clog2(DATA_WIDTH){1'b0}};
```

收满 8 bit → `byte_done` 脉冲 → `bit_cnt` 归零 → 下一个 `sample_stb` 开始下一字节。

`spi_system_interface` 在 `byte_done && cs_active` 时加载下一 TX 字节，不需要等 CS 再拉低一次。

### 2.3 帧结束：`cs_end`

```54:56:spi_slave/spi_slave_p1/src/spi_shift_register.v
            if (cs_end) begin
                bit_cnt  <= {$clog2(DATA_WIDTH){1'b0}};
                rx_shift <= {DATA_WIDTH{1'b0}};
```

CS 拉高 = 主机说「这帧结束了」。此时要：

1. **丢掉未完成的半字节**（只收了 4 bit 就结束 → 整个 RX 作废）
2. **复位 bit 计数和 RX 移位寄存器**

testbench T4 专门测这个：

```
发 0xDE 完整 8 bit，再发 0xAD 的前 4 bit，然后 CS 拉高
→ 0xAD 只传了一半，必须丢弃，不能误报 rx_valid
```

`cs_end` 时 `tx_valid` 也会清掉（`spi_system_interface` 第 37-38 行），表示这帧的发送请求结束。

### 为什么移位寄存器要接 `cs_*`？

如果没有这些信号，你只能「数满 8 个 bit 就算一字节」——无法处理：

- CS 中途拉高（半字节）
- 新帧开始时 bit 计数不从 0 起
- CS 无效期间误响应 stray 的 SCLK

## 3. 全双工：同一时刻收一个字节、发另一个字节

### SPI 是全双工的

每个 SCLK 周期，**MOSI 和 MISO 同时在传**。主机发 bit 的同时，从机也在回 bit。

以 echo 测试为例，主机发 `0x55`：

```
时刻          主机 MOSI (RX)     从机 MISO (TX)
─────────────────────────────────────────────
CS↓ 后        (还没开始)         发 tx_shift 的 MSB（比如 0x00 的 bit7）
第1个上升沿   收 B7 of 0x55      MISO 仍是 T7（上升沿前要保持稳定）
第1个下降沿   —                  移位，MISO 变成 T6
第2个上升沿   收 B6 of 0x55      ...
...
第8个上升沿   收 B0，byte_done    —
```

关键：**这一拍里 RX 在收 `0x55`，TX 在发 `0x00`（或上一轮 echo 值）——两个字节不同。**

### 所以需要两根移位寄存器

```24:25:spi_slave/spi_slave_p1/src/spi_shift_register.v
    reg [DATA_WIDTH-1:0] rx_shift;
    reg [DATA_WIDTH-1:0] tx_shift;
```

| 寄存器 | 驱动信号 | 数据来源 | 输出 |
|--------|----------|----------|------|
| `rx_shift` | `sample_stb` | `mosi_s` 移入 | 满 8 bit → `rx_byte` |
| `tx_shift` | `shift_stb` | `tx_load` 并行加载 | `tx_bit` → `miso` |

它们**移位时机不同**（上升沿 vs 下降沿），**数据内容不同**（收到的 vs 要发的），没法共用一根寄存器。

### 读 TX 路径

加载（并行写入整字节）：

```61:63:spi_slave/spi_slave_p1/src/spi_shift_register.v
            if (tx_load) begin
                tx_shift <= tx_byte;
            end
```

移出（每 bit 一个 `shift_stb`）：

```37:39:spi_slave/spi_slave_p1/src/spi_shift_register.v
    wire [DATA_WIDTH-1:0] tx_shift_next = LSB_FIRST
        ? {1'b0, tx_shift[DATA_WIDTH-1:1]}
        : {tx_shift[DATA_WIDTH-2:0], 1'b0};
```

输出当前最高位（MSB first）：

```42:42:spi_slave/spi_slave_p1/src/spi_shift_register.v
    assign tx_bit  = LSB_FIRST ? tx_shift[0] : tx_shift[DATA_WIDTH-1];
```

`miso` 在 controller 里组合输出：

```75:75:spi_slave/spi_slave_p1/src/spi_controller.v
    assign miso = cs_active ? tx_bit : 1'b0;
```

`tx_load` 后**立刻**能在 MISO 上看到首 bit（不用等第一个 `shift_stb`），满足 Mode 0「第一个上升沿前 MISO 已稳定」的要求。

### 读 RX 路径

```29:31:spi_slave/spi_slave_p1/src/spi_shift_register.v
    wire [DATA_WIDTH-1:0] rx_shift_next = LSB_FIRST
        ? {mosi_s, rx_shift[DATA_WIDTH-1:1]}
        : {rx_shift[DATA_WIDTH-2:0], mosi_s};
```

MSB first 时：每次 `sample_stb` 把 `mosi_s` 从低位挤进去，8 次后拼成完整字节。

满 8 bit 时锁存到 `rx_byte_r` 并打 `byte_done`（因为移位过程中 `rx_shift` 是「半成品」，用户要的是完整字节）。

## 4. 把一整字节从头到尾串起来

以 **Mode 0、MSB first、主机发 `0x55`、从机 echo 发 `0x00`** 为例：

```
① CS 下降沿
   cs_start=1 → bit_cnt=0
   tx_load=1 → tx_shift=0x00 → miso=tx_shift[7]=0

② 第1个 SCLK ↑（sample_stb）
   rx_shift 移入 mosi(0x55 的 bit7=0)
   bit_cnt: 0→1

③ 第1个 SCLK ↓（shift_stb）
   tx_shift 右移 → miso 变成下一位

④ 重复 ②③，bit_cnt 到 7

⑤ 第8个 sample_stb
   rx_shift 收齐 → rx_byte_r=0x55, byte_done=1
   bit_cnt 归零

⑥ byte_done 后（CS 仍低）
   spi_system_interface: rx_valid=1, rx_data=0x55
   若 tx_ready: tx_load 下一字节（echo 的 0x55）到 tx_shift
```

如果 CS 一直不低，回到 ② 开始下一字节，**没有新的 `cs_start`**。

## 5. 对照表：看到信号就知道在干什么

| 你看到的代码 | 实际在回答的问题 |
|-------------|-----------------|
| `sclk_sync` 双 flop | SPI 引脚和 FPGA 不在同一时钟域，要先同步 |
| `sample_stb` / `shift_stb` | 今天是上升沿采样还是下降沿移位？（mode 问题） |
| `cs_active` | 现在算不算在一帧里？ |
| `cs_start` | 新帧刚开始，计数要清零、首字节要加载 |
| `cs_end` | 帧结束了，半字节丢掉 |
| `rx_shift` + `sample_stb` | 从 MOSI 收 bit |
| `tx_shift` + `shift_stb` | 往 MISO 发 bit |
| `tx_load` | 并行装入下一个要发的字节 |
| `byte_done` | 刚收满 8 bit，通知用户逻辑 |
| `rx_byte_r` 单独锁存 | 移位中是半成品，完成后才给用户 |

## 6. 为什么你的简化版会「感觉该有但没有」

你的 `spi_shift_register_rx` 思路是：

> 一个计数器 + 一位一位写进寄存器，满 8 个就 flag

这个模型本身没错，但少了三层包装：

1. **没有 `sample_stb`** → 不知道何时该移（mode / SCLK 问题）
2. **没有 `cs_*`** → 不知道帧边界（多字节、半字节丢弃）
3. **只有 RX** → 全双工还要平行的 TX 路径和 `tx_load`

加上这三层，代码就从「教学用移位器」变成「能过你们 T1–T5 测试的 SPI Slave」。

## 7. 一句话总结

- **Mode 解耦**：controller 把 CPOL/CPHA 翻译成 `sample_stb` / `shift_stb`，移位寄存器只管「收到脉冲就移」。
- **CS 帧**：`cs_start` 开新帧、`cs_active` 标记帧内、`cs_end` 收尾并丢弃垃圾 bit。
- **全双工**：同一 SCLK 周期 MOSI/MISO 同时在传，且收/发边沿不同 → 必须 `rx_shift` 和 `tx_shift` 各干各的。

如果你愿意，下一步可以只盯 **一个 bit 周期**，我把 `sample_stb`、`shift_stb`、`rx_shift`、`tx_shift`、`miso` 在同一拍的变化按 clk 周期列成表，对着仿真波形看会更直观。
