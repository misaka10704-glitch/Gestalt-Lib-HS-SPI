# lab_sys：连续捕捉 → SPI 比对 → 单帧显象

## 目标

排除「跨帧拼接假毛刺」，用 **adc vs SPI 收回** 的错字节数判断是否真有 SPI/CDC 问题。

## 链路

```
连续采 N=128 点（adc_clk 1MHz，时间连续）
  → SPI Master 整段发出 → 片内回环 → Slave 收回
  → 逐字节比对 → ERR
  → UART 一帧上送（含 ERR + DATA）
  → host 默认只画「最新一帧」
```

## 帧格式（注意多了 ERR）

```
AA 55 | SEQ | N | ERR | DATA[0..N-1] | XOR
```

`XOR = SEQ ^ N ^ ERR ^ DATA[…]`  
`ERR` = 本拍 `adc_buf[i] != spi_rx_buf[i]` 的个数（≤255）

## LED

| LED | 含义 |
|-----|------|
| 0 | busy |
| 1 | 心跳 |
| 2 | 曾出现过 SPI 错（粘滞） |
| 3 | 最近一拍 ERR≠0 |

## 上板

1. 顶层必须是 **`lab_sys`**
2. 综合下载
3. 关 WCHSerial 后：

```bash
cd host
python3 lab_sys_plot.py -p /dev/cu.usbserial-20230814131 --xlen 128
```

看状态行：`spi_err=…`。若曲线毛刺且 `spi_err>0`，才是 SPI/移位问题；若曲线光滑且 `spi_err=0`，毛刺曾是拼接假象。

加压：改 `lab_sys.v` 里 `.SPI_CLK_DIV(2)`（更快 sclk）再综合。

旧拼接行为（不推荐）：加 `--concat`。
