# lab_sys_ext：飞线非理想 SPI

Master 的 **SCLK / MOSI** 打到管脚，Slave 从邻脚收回；用飞线模拟板级延迟/干扰。

## 接线（两根飞线）

正面顶排 **FPGA 排针**（见 `Gowin/逻辑派功能框图-T.png`），连续四脚：

```
… PB5 | G15 | G14 | G16 | H15 | L9 …
         │     ▲     │     ▲
         └──飞线┘     └──飞线┘
```

| 从（Master 出） | 到（Slave 入） | 板脚 |
|----------------|----------------|------|
| `spi_sclk_o` | `spi_sclk_i` | **G15 → G14** |
| `spi_mosi_o` | `spi_mosi_i` | **G16 → H15** |

两根杜邦线各跨邻脚即可。`CS` 仍在 FPGA 内直连。

## 怎么复现错误

1. 打开 `lab_sys_ext.gprj`，顶层 **`lab_sys_ext`**
2. 综合下载
3. **先不插飞线**：应 `spi_err` 很大 / LED2 亮，曲线乱
4. **插上两根飞线**：短接良好时 `spi_err→0`，曲线光滑
5. 加压：加长飞线、靠近干扰源，或改 `.SPI_CLK_DIV(2)` 再综合

上位机与理想版相同（帧含 ERR）：

```bash
cd host
python3 lab_sys_plot.py -p /dev/cu.usbserial-20230814131 --xlen 128
```

看 `spi_err=` / `err_frames=`。
