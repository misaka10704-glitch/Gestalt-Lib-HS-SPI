# spi_verify 测试：片内 SPI 主从 + FIFO（逻辑派 G1）

初级上板目标：不接外部 MCU/ADC，在 FPGA 内用 **Master ↔ Slave 回环** 验证移位、字节完成、`async_fifo` / `spi_fifo_interface` 链路。

本目录为 **工程/约束/说明**；可综合 RTL IP 在仓库**首目录**。

参考资料：`../../Gowin/逻辑派引脚分配表.pdf`、`../../Gowin/立创逻辑派FPGA_G1_IO.cst`、`../../internal/Research/Theory(1).pdf`。

## 2. 系统组成

```
key[1] 按下
   → 填 Master TX FIFO（0x10,0x11,…）
   → spi_master 发一帧
   → 片内 SPI 线 → spi_slave_controller → spi_slave_shift
   → Slave RX FIFO 应收 0x10..
   → Slave TX 固定回波 0x5A → Master RX FIFO 应收全 0x5A
   → 比对 → led pass / fail
```

| 角色 | 路径 |
|------|------|
| 板级顶层（IDE 选 **spi_verify**） | `projects/spi_verify/spi_verify.v` |
| 验证核心 | `spi_verify_top.v`（首目录） |
| 主机/从机/FIFO | 首目录各 `.v` |
| 约束 | `projects/spi_verify/spi_verify.cst` |

默认参数：`LOOPBACK=1`，`USE_PLL=0`，`CLK_DIV=8`，`XFER_N=8`，Mode0。

## 3. 板卡与管脚（逻辑派 FPGA-G1）

| 顶层信号 | 管脚 | 板载功能 |
|----------|------|----------|
| `sys_clk` | T7 | 系统时钟 |
| `key[0]` | D11 | `rst_n`（低有效） |
| `key[1]` | F10 | 启动自测 |
| `led[0]` | R9 | busy |
| `led[1]` | C10 | pass |
| `led[2]` | R7 | fail |
| `led[3]` | N6 | overflow 粘滞 |
| `spi_sclk` | L12 | ↔ARM SPI0_SCK |
| `spi_mosi` | L16 | SPI0_MOSI |
| `spi_miso_ext` | L13 | SPI0_MISO |
| `spi_cs_n` | E15 | SPI0_NSS |

PLL：封装内暂绑 `sys_clk` / `locked=1`；要用 PLL 时在封装里接 IP 并 `USE_PLL=1`。

## 4. Gowin IDE 工程要点

1. 打开本目录 **`spi_verify.gprj`**（已含首目录 IP 相对路径）。
2. 顶层模块设为 **`spi_verify`**。
3. 约束为 **`spi_verify.cst`**。
4. 综合 → P&R → 下载。

若 LED 极性相反，在 `spi_verify.v` 里对 `led[*]` 取反。

---

## 5. 上板操作

1. 勿按住 `key[0]`（复位）。
2. 按 `key[1]`：busy → 结束看绿(pass)/蓝(fail)；LED1 红为 overflow 粘滞。
3. 可用逻辑分析仪在 L12/L16/L13/E15 观察 SPI。

## 6. 通过判据

| 检查项 | 期望 |
|--------|------|
| Slave RX | `0x10 … 0x10+N-1` |
| Master RX | 全 `0x5A` |
